搭载SpringMvc环境

  • web.xml

SpringMvc的DispatcherServlet是一个前端控制器，拦截URL后，通过HandlerMapping获得HandlerExecutionChain（拦截器+controller），做的是一个请求分发的工作(handler),接下来是拦截器和controller工作，处理器完成后，会向DispatcherServlet返回一个ModelAndView（模型数据和逻辑视图名），DispatcherServlet会根据逻辑视图名，选择相应的视图（ViewResolver），传入模型数据进行渲染，最终将渲染结果返回client。

  • viewResolver：视图解析器，主要作用就是根据视图名（ViewName），返回视图 （View）

  • InternalResourceView其中的一种实现：主要是jsp的渲染展示，前缀/后缀修饰url后，变成/WEB-INF/views/hello.jsp(假如是”hello”)

ModelAndView和return字符串、请求和转发

请求和转发

  • redirect:重定向
    • 相当于用户重新发起另外一个新的http请求,浏览器上的url会变化
    • /WEB-INF/下的jsp受到安全保护，无法直接访问这些视图
  • forward:转发
    • Web容器内部进行转发,url上看不出来，是服务端在继续进行。
    • 通过转发的方式是可以访问/WEB-INF/下的jsp

相对路径和绝对路径

  • 项目：/
  • 访问controller的某一个handler映射为/hello1/test
  • 完整url：(http://localhost:8080/hello1/test)
  • 相对路径：" return "forward:hello";"，结果为(http://localhost:8080/hello1/hello),相对当前访问路径"/hello1"的路径
  • 绝对路径：" return "forward:/hello";"结果为(http://localhost:8080/hello)，是相对当前项目"/"的根路径
